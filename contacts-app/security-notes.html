Securuty Notes

An HTTP Client will not always be a browser. Before such a situation, it is necessary to keep in mind the pertinent rules and security measures.

Therefore, we can not be naive and believe that the only ones who are going to add contact to our application will be the users with access to the Application, with commands such as (CURL) they can make requests of type (POST) fron the console

Example

Step # 1

By console we access with (curl) to the path of a php file 


    windo@w-dev MINGW64 /c/xampp/htdocs/myPHPcode/php-mastermain/contacts-app (master)
    $ curl http://localhost/myPHPcode/php-mastermain/contacts-app/contacts-add.php

This comand will throw the html code of that page By console

Step # 2

If we pass the request the request type, in this case "POST" (-X POST), an Header which indicates the type of content. this comes in every HTTP request (--header "Content-Type: application/x-www-form-urlencoded") a Data, in this case the contact to create (--data "name=CURL&phone_number=45454545454" ) 

    curl -X POST --header "Content-Type: application/x-www-form-urlencoded" --data "name=CURL&phone_number=45454545454" http://localhost/myPHPcode/php-mastermain/contacts-app/contacts-add.php
